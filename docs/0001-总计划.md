# DeployLab：部署训练靶场


## 一、项目定位

一句话：

> **DeployLab 是一个“服务器部署练习项目”，用最简单的业务，把前端、后端、数据库、缓存、Nginx、Docker Compose 全链路串起来。**

不要做复杂业务，核心目标是：

```txt
我能不能把一个完整项目从 0 部署到服务器？
我能不能知道每个请求走到哪里？
我能不能在服务挂了之后查日志修复？
我能不能更新前端、更新后端、保留数据库数据？
```

---

## 二、做什么项目

做一个简单的「学习任务管理系统」，名字叫 `DeployLab 学习任务板`。

功能就 6 个：

```txt
1. 新增学习任务
2. 查看任务列表
3. 修改任务状态
4. 删除任务
5. 查看服务健康状态
6. 查看 Redis 访问计数
```

具体的功能、接口、表、页面设计见下面第五 ~ 七章。

---

## 三、为什么这个项目适合你？

因为它刚好覆盖部署里的关键链路。

### 1. Vue 前端

练：

```txt
npm run build
dist 静态文件
Nginx 托管
/api 代理
前端环境变量
```

### 2. Express 后端

练：

```txt
Node 服务启动
接口路由
连接 MySQL
连接 Redis
读取环境变量
容器日志
```

### 3. MySQL

练：

```txt
建库
建表
数据持久化
volume
容器重启后数据是否还在
```

### 4. Redis

练：

```txt
缓存服务
访问计数
服务依赖
容器间通信
```

### 5. Nginx

练：

```txt
托管前端
反向代理 /api
SPA fallback
统一入口
只暴露 80 / 443
```

### 6. Docker Compose

练：

```txt
多容器编排
服务名通信
depends_on
network
volume
restart 策略
环境变量
```

## 四、最终部署结构

```txt
Ubuntu 本机服务器
└─ Docker Compose：deploy-lab
   ├─ frontend-nginx 容器
   │  ├─ Nginx
   │  ├─ Vue dist
   │  └─ /api → server:3000
   │
   ├─ server 容器
   │  ├─ Express
   │  ├─ 连接 mysql:3306
   │  └─ 连接 redis:6379
   │
   ├─ mysql 容器
   │  ├─ deploy_lab 数据库
   │  └─ mysql_data volume
   │
   └─ redis 容器
      └─ 缓存访问计数
```

请求流向：

```txt
浏览器 → 服务器IP:80 → frontend-nginx ┬─ 返回 Vue 静态文件
                                      └─ /api 代理 → server:3000
                                                       ├─ mysql:3306
                                                       └─ redis:6379
```

## 五、接口设计

后端 Express 做这些接口就够了：

```txt
GET    /api/health
GET    /api/tasks
POST   /api/tasks
PATCH  /api/tasks/:id/status
DELETE /api/tasks/:id
GET    /api/stats
```

含义：

```txt
GET /api/health
  检查后端是否正常

GET /api/tasks
  获取任务列表

POST /api/tasks
  新增任务

PATCH /api/tasks/:id/status
  修改任务状态

DELETE /api/tasks/:id
  删除任务

GET /api/stats
  返回 Redis 里的访问次数
```

## 六、数据库设计

MySQL 只需要一张表。

```sql
CREATE TABLE tasks (
  id INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(255) NOT NULL,
  status VARCHAR(50) NOT NULL DEFAULT 'todo',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

任务状态就三个：

```txt
todo
doing
done
```

够用了。

## 九、你应该分阶段做,不要一次做完

### 第 1 阶段：本地开发跑通

目标：

```txt
Vue 本地 dev server
Express 本地启动
MySQL 容器启动
接口能读写数据库
```

先不管 Nginx。

---

### 第 2 阶段：后端 Docker 化

目标：

```txt
server 做成镜像
Express 在容器里跑
能连接 MySQL 容器
```

重点理解：

```txt
DB_HOST 不能写 localhost
应该写 mysql
```

---

### 第 3 阶段：Compose 编排 server + mysql

目标：

```txt
docker compose up -d
server 和 mysql 一起启动
```

练：

```bash
docker compose ps
docker compose logs server
docker compose exec mysql mysql -uroot -p
```

---

### 第 4 阶段：加入 Redis

目标：

```txt
server 连接 redis
/api/stats 返回访问次数
```

练：

```txt
server → redis:6379
```

---

### 第 5 阶段：加入前端和 Nginx

目标：

```txt
访问服务器 IP
看到 Vue 页面
/api 自动代理到后端
```

重点理解：

```txt
前端容器本质是 Nginx + dist
```

---

### 第 6 阶段：发布更新练习

你故意改代码：

```txt
前端改一个标题
后端改一个接口返回
数据库新增一个字段
```

然后练：

```bash
docker compose build frontend
docker compose up -d frontend

docker compose build server
docker compose up -d server
```

---

### 第 7 阶段：故障排查练习

故意制造错误：

```txt
1. DB_HOST 写错
2. MySQL 密码写错
3. Nginx proxy_pass 写错
4. server 容器不启动
5. 删除 dist
6. 删除 volume
7. 端口 80 被占用
```

然后用日志排查。

## 十、项目验收标准

你做到下面这些，就算合格：

```txt
1. 在服务器执行 docker compose up -d 后项目能启动
2. 浏览器访问服务器 IP 能看到前端
3. 前端能调用 /api/tasks
4. 后端能读写 MySQL
5. MySQL 数据重启容器后不丢
6. 只有 80 端口对外开放
7. server/mysql 不直接暴露公网
8. docker compose logs 能看到日志
9. 改前端后能重新 build 发布
10. 改后端后能重新 build 发布
```
